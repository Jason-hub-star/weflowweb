## 2. Brand Definition

### 2.1 브랜드 의미

WEFLOW는 `WE`와 `FLOW`의 결합이다.

- WE: 우리, 사람, 관계, 함께하는 가치
- FLOW: 흐름, 성장, 연결, 앞으로 나아가는 움직임

브랜드 해석:

- 사람과 기술이 함께 흘러가며 더 좋은 방향을 만드는 회사
- 단순히 개발만 하는 회사가 아니라, 기술은 뒤에서 받쳐주고 사람은 앞에서 빛나게 하는 흐름
- 혼자 성공하는 회사보다 함께 흘러가며 성장하는 회사를 지향

### 2.2 브랜드 슬로건 후보

우선순위 1:

```text
WEFLOW
Flow Together, Grow Beyond.
```

보조 후보:

```text
WEFLOW
People move. Technology follows.
```

### 2.3 핵심 포지셔닝

WEFLOW는 단순 홈페이지 제작사가 아니라, 문의 전환 구조를 설계하고 제작 후 운영까지 관리하는 회사다.

핵심 문장:

```text
문의로 이어지는 홈페이지를 만듭니다
```

보조 문장:

```text
홈페이지 제작부터 광고 연동·운영 관리까지
단순 제작이 아닌 문의 구조까지 설계합니다
```

차별화 문장:

```text
제작부터 운영 · 광고 · 관리까지 한 번에
```

## 3. Problem

소상공인과 소규모 사업자는 홈페이지가 필요하지만 다음 문제를 겪는다.

- 어떤 구조로 만들면 문의가 늘어나는지 모른다.
- 제작사, 광고 대행, 블로그/인스타 운영, SEO 등록을 따로 맡겨야 해서 복잡하다.
- 홈페이지 제작 후 관리가 끊겨 실제 문의 개선으로 이어지지 않는다.
- 네이버/구글/당근/카카오/인스타 등 유입 채널을 홈페이지와 연결하지 못한다.
- 사이트는 있으나 CTA, 모바일 동선, 문의폼, 사례, 후기, 검색 등록이 약하다.

## 4. Goal

이번 개편의 목표는 다음과 같다.

- 첫 화면에서 WEFLOW가 무엇을 하는 회사인지 5초 안에 이해시킨다.
- 모든 핵심 화면에서 무료진단/문의/예약으로 이동할 수 있게 만든다.
- 제작 플랜, 케어 플랜, 광고 플랜을 명확히 구분해 가격 신뢰도를 높인다.
- 사례/후기/프로세스/진단 항목을 통해 문의 전환을 유도한다.
- 문의폼과 예약폼 데이터를 관리자에서 확인하고 상태 관리할 수 있게 한다.
- 모바일에서도 하단 고정바를 통해 상담, 카카오톡, 블로그, 무료진단 접근이 가능하게 한다.

## 9. Open Design Brief

이 문서의 목적은 콘텐츠/기능 요구사항을 고정하는 것이다.  
색상, 카드 형태, 배경 질감, 아이콘 스타일, 섹션 여백, 그래픽 구성 같은 세부 시각 표현은 이 단계에서 확정하지 않는다. 디자인 AI는 아래 필수 요구사항을 유지한 상태에서 2~3개의 시각 방향을 먼저 제안해야 한다.

디자인 AI에게 전달할 brief:

```text
Create a conversion-focused Korean service website for WEFLOW, a web production and care-plan company.
The site must feel like a premium but practical growth console for small business owners.
Primary goal: increase free diagnosis requests, reservations, and consultation clicks.
Tone: trustworthy, fast, modern, data-aware, operational, not decorative.
Visual direction is not fixed yet. First propose 2-3 distinct directions, then design the selected one.
Required feeling: trustworthy, conversion-focused, modern, readable, mobile-friendly, practical for small business owners.
Avoid medical/healthcare technology imagery or claims.
Do not create a generic SaaS landing page. Build the actual service website with pricing, cases, forms, sticky consultation UI, and admin requirements.
```

추천 Open Design skill 조합:

- `web-prototype`: 전체 공개 사이트 시안
- `saas-landing`: 홈/랜딩페이지 전환 구조
- `pricing-page`: 제작 플랜/케어 플랜/광고 플랜 가격 화면
- `dashboard`: 관리자 사이트
- `critique`: 최종 시안 자가 검수

## 10. Visual Direction

이 섹션은 확정 디자인이 아니라 디자인 탐색 기준이다.  
디자인 AI는 기존 웹사이트 스타일에 갇히지 말고, 아래 조건을 만족하는 서로 다른 방향 2~3개를 먼저 제안한다. 최종 컬러/카드/배경/아이콘/그래픽 스타일은 선택 후 확정한다.

고정하는 것:

- 브랜드명, 핵심 문구, 가격, 플랜, 폼 필드, 페이지 구성, 버튼 동작, 금지사항
- 전환 중심의 정보 구조
- 모바일에서 문의하기 쉬운 구조
- 글자 끊김/겹침 없는 반응형

열어두는 것:

- 정확한 컬러 팔레트
- 카드 모양과 그림자 강도
- 배경이 밝은지 어두운지
- hero 그래픽 구성
- 아이콘 스타일
- 섹션 여백과 레이아웃 리듬
- 애니메이션 방식

### 10.0 디자인 방향 제안 요구

디자인 AI는 바로 한 가지 시안을 만들지 말고, 먼저 아래와 같은 방향 3안을 제안한다.

1. Tech Utility / 데이터 중심
   - 장점: 전환 측정, 운영 관리, 관리자 기능과 잘 맞음
   - 주의: 너무 차갑거나 SaaS 템플릿처럼 보이지 않게 조정
2. Premium Minimal / 신뢰 중심
   - 장점: 가격 신뢰도, 전문성, 깔끔한 문의 동선에 강함
   - 주의: 밋밋하거나 차별점이 약해지지 않게 사례/CTA 강조
3. Local Business Growth / 소상공인 친화 중심
   - 장점: 대표들이 쉽게 이해하고 문의하기 쉬움
   - 주의: 너무 가벼운 광고 전단처럼 보이지 않게 품질감 유지

각 방향은 다음 기준으로 비교한다.

- WEFLOW 브랜드와 어울리는가
- 소상공인 대표가 신뢰할 수 있는가
- 무료진단 CTA가 잘 보이는가
- 가격/케어플랜이 읽기 쉬운가
- 모바일에서 문의 동선이 짧은가
- 기존 사이트보다 새롭고 명확한가

### 10.1 전체 톤

- 소상공인이 신뢰할 수 있는 실무형/운영형 느낌
- “문의 증가”, “빠른 제작”, “운영 관리”가 바로 읽히는 정보 구조
- WEFLOW 로고와 어울리는 색을 사용하되, 정확한 팔레트는 시안 단계에서 제안
- 과장된 미래형, 과한 장식형, 흔한 템플릿형 랜딩페이지 느낌은 피한다.

### 10.2 디자인 키워드

- 신뢰감
- 빠른 실행
- 전환 중심
- 운영 관리
- 데이터/성과
- 모바일 문의 동선

### 10.4 타이포그래피

- 한국어 가독성 우선
- `Pretendard`, `Noto Sans KR`, 또는 repo의 현재 `Geist` + fallback 가능
- hero headline은 줄바꿈을 직접 제어한다.
- 모바일에서 단어가 끊기지 않도록 `word-break: keep-all` 기본 적용
- 버튼/카드 안 텍스트는 2줄 이내로 자연스럽게 줄바꿈

### 10.5 모션

- 후기 슬라이더는 자동 흐름, hover/focus 시 정지
- CTA 버튼은 충분히 눈에 띄어야 한다. glow/shadow 여부와 강도는 디자인 방향에 맞게 제안한다.
- 섹션 진입 애니메이션은 과하지 않게 opacity/translate
- 폼 입력 중 레이아웃 shift 금지

## 29. Risks

1. 범위 과다
   - 공개 사이트 + 관리자 + DB까지 포함되어 MVP가 커질 수 있다.
   - 해결: 공개 사이트 우선, 관리자 MVP는 문의/예약 목록 + 상태변경 + CSV부터.

2. 가격 정보 충돌
   - WEFLOW CARE 가격이 289,000원과 339,000원으로 다르게 언급됨.
   - 해결: 구현 전 최종 가격 확인.

3. 외부 링크 미정
   - 카카오, 블로그, 인스타, 사례 블로그 URL이 필요하다.
   - 해결: 임시 `#TODO_REAL_LINK`로 구현 후 배포 전 교체.

4. SEO 표현 리스크
   - 상단등록/상단노출 표현이 보장처럼 보일 수 있다.
   - 해결: 약관과 안내문에서 보장 아님을 명확히 하거나 표현 조정.

5. 관리자 보안
   - 단순 비밀번호 hardcode 금지.
   - 해결: Supabase/Firebase Auth 또는 서버 세션 적용.
