## 4. 유지/참고 방향

유지하고 싶은 것:

- 브랜드/스토리 페이지
- 제품 상세 페이지 구성
- 이미지/톤앤매너

닮고 싶은 포인트:

- 상단 메뉴 구조: 스토리/제품/블로그/후기/공지
- 상품 리스트/카드 UI
- 후기 섹션
- 전체 분위기: 프리미엄/미니멀

원하는 분위기:

- 밝고 친근
- 프리미엄/미니멀
- 30~40대 타깃

컬러 방향:

- 기존 컬러 유지
- 다크톤(블랙/차콜)

싫어하는 스타일:

- 기타로 선택됨. 구체 텍스트 없음.

## 5. 홈/진입 구조

메인 페이지에서 강조할 것:

- 제품 구매(바로 구매 유도)
- 이벤트/프로모션

해석:

- 쇼핑/결제 기능은 필요 없다고 응답했으므로 `제품 구매`는 실제 결제가 아니라 `플랜 선택 → 문의/견적/카카오 상담` 전환으로 해석한다.

메인 진입 구조:

- 캠페인/랜딩 중심
- 시즌별 교체 가능 구조 필요

## 6. 페이지/콘텐츠 구성

메뉴/페이지 구성:

- 스토리/브랜드 소개
- 제품(리스트)
- 제품 상세
- 제품 후기(리뷰)
- FAQ
- 고객센터/문의
- 기타

필요 페이지:

- 제품/서비스 소개
- 상세페이지
- 가격 안내
- 후기
- 사례
- 블로그
- 공지
- FAQ
- 문의

성분/기능성 정보:

- 지금처럼 상세하게(페이지 여러 개)
- 요약형으로 정리(페이지 수 줄이기)

해석:

- WEFLOW는 화장품/건기식 성분 사이트가 아니므로, 이 항목은 `서비스 기능/플랜 정보`를 상세 페이지와 요약 카드 양쪽에서 제공하는 요구로 변환한다.

## 11. 디자인 반영 요약

- Devfive식 고정 헤더와 카드/후기/프로세스 구조를 참고한다.
- 최신 설문 기준으로 SaaS 플랫폼/제품 카드/제품 상세/리뷰/블로그/공지/FAQ 구조를 보강한다.

- 캠페인/랜딩 중심 진입을 위해 hero/banner는 관리자에서 교체 가능한 전제로 설계한다.
